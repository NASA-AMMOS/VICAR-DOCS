jpl.mipl.tlm.ipc

Class Worker

  • java.lang.Object
    • jpl.mipl.tlm.ipc.Worker

  • public abstract class Worker
extends java.lang.Object

    Implementers of instrument product constructors should subclass Worker and do their processing in the processData() method.

    It is guaranteed that the framework will call:

    • init()
    • processData()
    • cleanup()
    in that order, once per client processing request. A single Worker object must be able to handle multiple (sequential) requests.

      • setSimulateTDSWaitFlag

        protected void setSimulateTDSWaitFlag(boolean simuTDSWait)
        This method is used to indicate if the recvSFDU is to call the actual method on the TDS with time outs. For example, if the recvSFDU is called with WAIT_FOREVER, but this flag is set to true, the worker will ask the Instrument_Service to simulate the WAIT_FOREVER. This will allow the Instrument_Service to catch CTRL-C signals and stop the service cleanly. By default, the simulation is preformed.
