      • recvSFDU

        SfduStatus recvSFDU(int timeOut,
                  byteSeqHolder sfdu)
        Returns an array of bytes from a telemetry data source (TDS). The sfud.buffer length will be jpl.mipl.tlm.TDSIF_Service.MAX_SFDU_SIZE, but only sfdu.length bytes will have been filled with data. After calling this method, check sfdu.length to find out how much data was put into the sfdu.buffer, and check the return value to find if there is more data at the data source.
        Parameters:
        timeOut - The amount of time to wait for an SFDU, or WAIT_FOREVER for no time out. Behavior is undefined for negative values other than WAIT_FOREVER;
        Returns:
        true if the telemetry data source has indicated that there is more data available.

      • isOkayToContinue

        boolean isOkayToContinue()
        While in Worker.processData(), this should be checked often. If it returns false, then this stream has been halted, so processing should cease. When this returns false, the Worker must return from Worker.processData().

      • edrAvailable

        void edrAvailable(java.lang.String fileName)
        The worker should call this method when it has finished an EDR. This will announce the availability of the EDR to Jedi and any other listeners which are registered.

      • setSimulationThreshHold

        void setSimulationThreshHold(int timeOut)
        Sets the simulation thresh hold value. If the timeOut supplied in call to recvSFDU is greater then this value, then the recvSFDU breaks every timeOut seconds indicated by this parameter. This allows checking for isOkayToContiue in some timely manner. Thus solving the problem of ignoring CTR-C when the instrument service is hanging in recvSFDU waiting for more data.
