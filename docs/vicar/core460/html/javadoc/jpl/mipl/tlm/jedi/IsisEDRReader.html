Class IsisEDRReader

    EDRReader

    public class IsisEDRReader
extends java.lang.Object
implements EDRReader
    An EDR Object assembler that ingests a Cassini ISS VIMS cube (for use in the new RealTime system Display Manager/Server). Basically, to use this object, you simply instantiate it with a ISIS file as input, stand back, and hope all goes well. You probably should not call this class directly. Use the EDRLoader class instead (it implements the EDRReader interface as well).

    Also, this object should handle EDR sized images fairly well. I can make no claims as to what will happen if you feed it a huge dataset.

    Lastly, this class was intended to offer a way for an EDR file to be made into a Java Object such that it could then be passed around through serialization/RMI mechanisms.

    NOTE: This implementation is really designed to read the Cassini VIMS EDRs as generated by Raj's Instrument Constructor. Since this is a multi-spectral EDR (there are 352 channels possible), we must understand the following caveat:
    We can only view 3 planes in the JEDI client
    Therefore, IsisEDRReader allows for the selection of three arbitrary bands to populate the resultant EDR object. This object is, therefore, much smaller and more efficient in transfer over the network to the subscribed JEDI clients. The default triplet is composed of the first three bands to obtain red, green, and blue, respectively.

      • setDNScaling

        public void setDNScaling(double choice)
        Description copied from interface: EDRReader
        Determines how the data should be scaled to 8-bit. Normally, the Reader will scale the data based upon discrete divisors. I.e.:
        max DNdivisor
        255 1
        511 2
        1023 4
        2047 8
        4095 16
        8191 32
        16383 64
        32767 128
        65535 256
        etc...
        This is designed to keep the histogram consistent with the dynamic range that the sensor is capable of. However, one may wish to have the image DNs scaled such that the maximum DN of the data is used to compute the scaling. This can cause the histogram to be much more filled. In this case, the histogram wouldn't be as correct, but likely more appealing.
