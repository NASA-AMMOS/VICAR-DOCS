jpl.mipl.io.streams

Class DataOutputStreamWrapper

  • java.lang.Object
    • java.io.OutputStream
      • jpl.mipl.io.streams.DataOutputStreamWrapper
  • All Implemented Interfaces:
    java.io.Closeable, java.io.Flushable, java.lang.AutoCloseable

    public class DataOutputStreamWrapper
extends java.io.OutputStream
    This class wraps an object implementing DataOutput into an OutputStream. Although most objects that implement DataOutput are already OutputStreams, not all are. Most notably, javax.media.imageio.stream.ImageOutputStream from the Image I/O package is this way.

    Only the three write() methods are used from the DataOutput object.

    It's unbelievable that this is not in java.io! And it's even more unbelievable that OutputStream is not an interface, thus avoiding the need for all this garbage.

    Since DataOutput provides absolutely nothing in the way of flushing or closing, it is assumed that the DataOutput object takes care of that on its own (or the application does it directly, outside of this wrapper). Thus the flush and close methods of this wrapper are no-ops. Thus this class is recommended as a last resort only.
