jpl.mipl.io.vicar

Class VicarBinaryLabel

  • java.lang.Object
    • jpl.mipl.io.vicar.VicarBinaryLabel

  • public class VicarBinaryLabel
extends java.lang.Object
    Manages a piece of Vicar Binary Label data, either the entire header or one line's worth of prefix. From this class, DataInput or DataOutput objects may be returned which allow you to read and/or write the data in the correct host representation.

    These DataInput and DataOutput objects all access the same internal buffer... to which you can also get direct access. It is up to the caller to synchronize any accesses to this buffer. It is possible to read something from the DataInput, modify it, and put it back via the DataOutput. However, each stream maintains its own pointer so you must write the same number of bytes as you read for them to stay in sync. And if the write gets ahead, the modified data will be seen by the read. In short, be careful about mixing modes.

        Returns an object that implements DataInput, which will provide access to the buffer via that interface. Data is translated according to the VicarDataFormat object, i.e. data in the buffer is in the (possibly foreign) format, while data returned by the DataInput object is always in Java format.
