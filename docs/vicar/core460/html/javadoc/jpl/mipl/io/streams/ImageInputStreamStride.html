jpl.mipl.io.streams

Class ImageInputStreamStride

  • java.lang.Object
    • jpl.mipl.io.streams.ImageInputStreamStride
  • Direct Known Subclasses:
    ImageOutputStreamStride

    public class ImageInputStreamStride
extends java.lang.Object
    This class implements the multi-element read functions of ImageInputStream but with the addition of an extra parameter, which specifies the pixel stride in the array. This allows one to read directly into an array (usually in a DataBuffer) where the pixels are not contiguous in memory. For example, a pixel interleaved array could be read from a band sequential file by reading into every 3rd array element.

    It would be nice to actually implement ImageInputStream (or extend ImageInputStreamImpl) in this class. However, to maintain JDK 1.3 compatibility, we can't do this (yet). That may be a future enhancement.

    An additional enhancement over ImageInputStream is that we can handle VAX-format reals in addition to forward and reverse IEEE reals. They are converted to Java reals (IEEE) as they are read in.

    This class tries to be as efficient as possible. For example, if the source object actually is an ImageInputStream and the pixel stride is 1, we call the ImageInputStream methods directly.

    Allowable input sources in order of preference are:

    • ImageInputStream
    • DataInput
    • InputStream
    Note that RandomAccessFile is a DataInput and thus is legal.

      • ImageInputStreamStride

        public ImageInputStreamStride(java.lang.Object stream,
                      int int_order,
                      int float_order)
        Constructor with different byte orders for ints and floats. 99.9999% of the time they'll be the same but since VICAR has different variables holding the ordering, so do we...

        A null stream is legal for use by subclasses like ImageOutputStreamStride, but not too useful elsewhere since there's no facility to re-set the stream object and any read call will fail with a NullPointerException.
