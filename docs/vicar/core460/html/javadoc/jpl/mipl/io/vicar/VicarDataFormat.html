jpl.mipl.io.vicar

Class VicarDataFormat

  • java.lang.Object
    • jpl.mipl.io.vicar.VicarDataFormat

  • public class VicarDataFormat
extends java.lang.Object
    This class handles all aspects of data in different host representation. It "knows" the data formats for each host type, and has factory methods for creating filter streams that translate pixels between Java internal representations, and the format required for a particular host.

    The supported integer data representations are:

    • HIGH - high byte first. Used by Java, Sun, SGI, etc.
    • LOW - low byte first. Used by Alpha and Intel architectures.

    The supported floating-point data representations are:

    • IEEE - IEEE floating-point format, high byte first. Java, Sun, SGI, etc
    • RIEEE - Reverse IEEE (low byte first). Intel and Alpha Unix.
    • VAX - VMS D/F floating point formats.

    Both integer and floating-point types may alternatively be specified using the strings NATIVE or LOCAL, which are equivalent. Both mean to use the "native" host representation for the machine on which you are running, which for Java is always HIGH/IEEE. If you Set using one of these, you'll Get either HIGH or IEEE, never NATIVE or LOCAL.

    The supported data types are:

    • BYTE - 8-bit unsigned data (Java byte, interpreted as unsigned)
    • HALF - 16-bit signed data (Java short)
    • FULL - 32-bit signed data (Java int)
    • REAL - 32-bit floating point data (Java float)
    • DOUB - 64-bit floating point data (Java double)
    • COMP - Complex data (two consecutive REAL's)
    • USHORT - 16-bit unsigned data (Java short, interpreted as unsigned) - not legal for VICAR but other formats use it.

    Valid host types are:

    • JAVA
    • SUN-SOLR
    • SGI
    • X86-LINUX
    • HP-700
    • VAX-VMS
    • AXP-VMS
    • AXP-UNIX
    • AXP-LINUX
    • SUN-3
    • SUN-4
    • X86-SOLR
    • ALLIANT
    • DECSTATN
    • MAC-AUX
    • MAC-MPW
    • TEK
    • NATIVE
    • LOCAL

      • getDataInputWrapper

        public java.io.DataInput getDataInputWrapper(java.lang.Object s)
        These "factory" methods will create and return a stream object that implements the DataInput or DataOutput interfaces. These streams may be private classes, but they are set up to do the correct data type translations given the current setup of this VicarDataFormat.

        It is quite possible that the factories may simply return the original stream, if it already implements DataInput or DataOutput and the data is already in Java format.

        There is no guarantee that the returned stream will support any given random-access or seek operations. For this reason, the original stream should be retained by the caller, and used to reposition within the file (if needed). The returned stream (if not simply the original) will only use the read() or write() functions of the given stream, so it will be able to make use of any repositionings done on the original stream.

        Throws:
        java.lang.IllegalArgumentException - if the provided object is not a subclass of InputStream or DataInput.
