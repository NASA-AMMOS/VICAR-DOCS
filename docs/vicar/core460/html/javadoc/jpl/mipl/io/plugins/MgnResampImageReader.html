      • readAsRenderedImage

        public java.awt.image.RenderedImage readAsRenderedImage(int imageIndex,
                                               javax.imageio.ImageReadParam param)
                                                 throws java.io.IOException
        This method returns a RenderedImage. This is useful in at least 2 situations: 1) The data for a RenderedImage is not grabbed until asked for by an application. A tiled image will only grab the tiles needed. In a file copy the whole image wouldn't need to be loaded into memory. It could be copied tile by tile. 2) The image has more than 3 bands. It can't be returned as a BufferedImage since no ColorModel exists for > 3 bands. The user can extract bands for display using ImageOps.
