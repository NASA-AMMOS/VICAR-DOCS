Class Connection

  • java.lang.Object
    • jpl.mipl.mars.mis.Connection

  • public class Connection
extends java.lang.Object
    Contains all of the methods and properties associated with a single websocket connection. It should be initialized once (and only once) for each incoming connection. It manages the list of files associated with the connection and communicates with the FileManager @see jpl.mipl.mars.mis.FileManager

    Nolan Miller

      • readMessage

        public void readMessage(java.lang.String data)
                 throws java.io.IOException
        Parses an incoming connection message and performs the appropriate actions and sends the appropriate response to the client. Valid operations are:
        • add: add a (comma separated list of) files using @see #add
        • remove: remove a (comma separated list of) files using @see #remove
        • get: get the key-value paired pixel value at a int,int index
        • get-raw: get the raw string value at a int,int index
        • get-keys get the list of band names for the files
        • get-list: get the list of band values for the files
        • get-mod get the list of mod times for the files
        • ls list the files currently associated with the connection
        • clear clear all files currently associated @see #clear
        • size returns the number of files currently associated
        • test prints a test statement out to the server log (stdout) for debugging only
        get: returns a map without standard order preserved but with key values for file paths.
        Parameters:
        data - the String to be interpreted
        Throws:
        java.io.IOException - If IO error occurs
