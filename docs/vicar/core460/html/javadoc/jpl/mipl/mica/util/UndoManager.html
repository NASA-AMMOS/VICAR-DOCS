Class UndoManager

  • java.lang.Object
    • jpl.mipl.mica.util.UndoManager
  • All Implemented Interfaces:
    java.beans.PropertyChangeListener, java.util.EventListener

    public class UndoManager
extends java.lang.Object
implements java.beans.PropertyChangeListener
    An UndoManager records PropertyChangeEvents it receives and can replay the changes back in reverse order. Each event is received, then placed on a Stack. When the undo() method is called, the manager pops the most recent event off the Stack and attempts to replay it in reverse. The source object is identified and then the set<PropertyName>(<oldValue>) method is identified. If such an accessor method can not be found for the event's property change, no action is taken. If the method is found, then the method is called using the event's oldValue property.

    Two special PropertyChangeEvent.propertyName values cause special behavior. The START_GROUPING and END_GROUPING events allow a series of events to be grouped into a single undoable change.

    NOTE: The event grouping is not safe for multi-threading.

    NOTE: It is the responsibility of objects that knowingly cannot undo events to either turn off logging or clear the stack. Methods are provided.
