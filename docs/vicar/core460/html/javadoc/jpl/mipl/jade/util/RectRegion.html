Class RectRegion

  • java.lang.Object
    • jpl.mipl.jade.util.RectRegion
  • All Implemented Interfaces:
    java.lang.Cloneable

    public class RectRegion
extends java.lang.Object
implements java.lang.Cloneable
    A class that consists of a collection of Rectangles. The rectangles may touch each other but they may not overlap.

    One use of this class is to maintain a "damage list" of rectangular areas that need to be repainted on a display. As each tile is repainted, it can be subtracted from the damage list.

    This functionality was largely lifted from the XvicRegion code in VICAR's Image Widget.

    Although Rectangle derives from Rectangle2D, we base this in Rectangle to allow direct, fast access to the x/y/width/height fields (rather than go through access functions). This requires some conversions from Rectangle2D to Rectangle at various places which would not be otherwise needed if we implemented to the Rectangle2D spec.

    TBD: This class "should" implement Shape in order to be more generally useful. However, this is not currently done, because the getPathIterator() functions were non-trivial and unnecessary for the intended use. If a Shape is needed, these functions could be implemented.

    Author:
    Bob Deen, JPL

      • subtract

        public void subtract(java.awt.Rectangle rect)
        Subtracts a rectangle from the region, leaving behind only parts of the region that are outside the rectangle. The general idea here is that Y clipping is done first, then X. Any box that straddles the edge of the rectangle is split into two. The first is completely clippd; the second is left until its turn in the loop to be clipped.
