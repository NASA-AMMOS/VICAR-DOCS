    public class LogServer
extends ActivatableServerBase
implements LogServer_IF
    The LogServer class is a RMI server that enables synchronized logging. The LogServer class handles multiple PGSLoggers simultaneously. Each PGSLogger (logger from now on) has a unique name; namely the absolute log file name where messages are logged. The createLogger() and removeLogger() methods dynamically creates and destroys logger objects respectively. The LogServer is not started by any of the PGS core servers. If a project decides to use the LogServer, the project needs to start the server on it's own. Our suggestion is to start the log server in one of the scheduler class's method. Following code snippet shows how to start the LogServer.

       PersistentStore_IF ps = jpl.mipl.pgs.utils.NamingUtils.getReferenceToPS();
       Map config = ps.getControllerConfig(this.pgsSession);
       if (config == null || config.size() == 0)
           throw new Exception("Could not get controller configuration");

       Map startupData = new Hashtable();
       startupData.put(jpl.mipl.pgs.perstore.Constants.MISSION, );
       startupData.put(jpl.mipl.pgs.perstore.Constants.USERID, "");
       startupData.put(jpl.mipl.pgs.perstore.Constants.PASSWD, "");
       startupData.put(jpl.mipl.pgs.perstore.Constants.SERVER_NAME, LogServer_IF.LOG_SERVER_NAME);
       startupData.put(jpl.mipl.pgs.perstore.Constants.SERVER_HOST,
                       config.get(jpl.mipl.pgs.perstore.Constants.SERVER_HOST));
       startupData.put(jpl.mipl.pgs.perstore.Constants.SERVER_PORT,
                       config.get(jpl.mipl.pgs.perstore.Constants.SERVER_PORT));
       ServerStarter ss = new ServerStarter();
       LogServer_IF logServer = (LogServer_IF) ss.startServer(startupData, this.LogServerClassName);
