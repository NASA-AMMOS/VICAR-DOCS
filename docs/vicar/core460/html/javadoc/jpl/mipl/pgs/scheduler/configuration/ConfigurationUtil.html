      • resolveScriptVariables

        public static java.lang.String resolveScriptVariables(java.lang.String input,
                                      java.util.Map context)
                                               throws java.lang.Exception
        Resolve script variables. Script variable is defined as "[]" or "#[]" meaning an expression [if ("X".equals.("X")) return "XON"; return "ZON";] (inline script) or a script name followed by parameters as "#[/home/rrp/scriptname:param1=value:param2=value]" It is assumed that all context variables are resolved in the input script. For example, if the original input was of the form [if ("%{myval}".equals("X")) return "XON"; return "ZON";] then %{myval} must have been substituted with proper value (e.g. using resolveContextVariables(String, Map)) before calling this method.
        Parameters:
        input - a string that only contains unresolved script calls.
        context - the context that is to be passed to the script. The context may be used by the script to look for variables in addition to what is passed along with command line.
        Returns:
        the input string with all references of "[]" or "#[]" replaced by the return value of evaluating the expression or script.
        Throws:
        if - the input is malformed. That is, it is missing a closing bracket or it contains embedded scripts or expressions. Or if the expression or script evaluation returns a null value.
        java.lang.Exception
        See Also:
        for details on the format of the script. Note that for inline script the arguments and the context are ignored.

      • processFromCfg

        public static void processFromCfg(org.w3c.dom.Element e,
                  java.util.Map<java.lang.String,java.lang.String> envs)
                           throws java.lang.Exception
        Process attribute , if present, of the given element e. The value of this attribute is a file that contains a well formed XML document whose root element is same as 'e' (that is, e.getNodeName().equals(document.getDocumentElement().getNodeName))). The XML document is used as base and 'e' is used as new definition of the base. That is, any changes to 'e' are applied to the document in file and it becomes new 'e'. This ConfigurationReader is used to read the XML file. The root element read from configuration file is also processed via this method. That is, if the XML document contains attribute, it is processed first and then the merged node is merged with original 'e'.
        Parameters:
        e - The node to merge with
        envs - The environment variables used to resolve value of attribute . Note that the envs does not get updated during the recursive calls to processFromCfg method. That is, the any environment variables defined in the XML file are not incorporated in to the original list of envs.
        Throws:
        java.lang.Exception - If the value of contains any unresolved references to environment variables.
