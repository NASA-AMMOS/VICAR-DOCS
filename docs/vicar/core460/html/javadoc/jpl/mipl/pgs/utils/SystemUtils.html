      • getEnv

        public static java.lang.String getEnv(java.lang.String var)
        Returns the value associated with the given environment variable. The environment variable are looked into two places: 1) Java Runtime Environment's System properties 2) Operating system's environment variables The name of the variable determines the lookup strategy. If the variable name starts with ENV. (e.g., ${ENV.HOME}), only OS variables are searched. Otherwise the method first looks into system properties for the variable, if not found, it uses OS specific commands to obtain value for the variable
        Returns:
        value associated with variable. null if variable is not defined, null if OS is not supported.

      • expandEnvs

        public static java.lang.String expandEnvs(java.lang.String pathname)
        Expands all environment variables in the given string. If a variable is undefined it is left alone. For example,
        • "this string will not be changed"
        • Given that $HOME maps to /user/home then the string "${HOME}" will be expanded to /user/home
        • ${HOME}/rrp/${OS_DIR}/t.txt will be expended to /home/rrp/linux/t.txt, given that $OS_DIR maps to linux.
        • file:${HOME}jplmipl will be expanded to file:/home/rrpjplmipl.
        • in the string "${HOME/some/${VICSYS}", "HOME/some/${VICSYS" will be interpreted as environment variable and will thus have unpredictable results.

        For precedence rules on environment variable lookups refer to getEnv
        Throws:
        java.lang.RuntimeException - if either the name of the environment variable contains characters used in Java regular expressions. Use proper escape characters to avoid runtime exception.

      • extractEnvs

        public static java.util.List<java.lang.String> extractEnvs(java.lang.String str)
        Extracts all environment variables and return it in the list. An environment variable is any string enclosed in "${" and "}".
        Returns:
        a list containing names of all environment variables used in the given string with out the enclosing braces and $ sign. For example, ${V2JAVA}/somedir/${MISSION} will return a list containing two values: V2JAVA and MISSION.
