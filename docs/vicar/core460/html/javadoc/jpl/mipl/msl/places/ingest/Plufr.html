    Plufr - PLaces Update Frame manager Report

    This class reads the CSV version (i.e. after it goes through chill_dp_view -csv) of the FmReport data product. It extracts all relevant frame info and creates a file suitable for upload into PLACES (which is typically done via "curl").

    The supplied filename should be that of the CSV file. Multiple data products can be processed at once (this is more efficient when doing the actual upload).

    This class must be as bulletproof as possible, so it can be called from MATIS directly and not affect the MATIS JVM. Alternatively, a main() function exists, which can be used instead and in any case acts as a test harness.

    Author:
    Bob Deen, JPL

        This function implements the functionality of Plufr. Inputs are a list of filenames (.csv files, the output of chill_dp_view -csv on a FmReport data product), and an output filename to be written to. Unlike the main() driver, the output is not tested and will be overwritten if it already exists. If the minSclk or maxSclk parameters are non-null, they will be compared against the actual sclk. If the sclk is outside the range, the sclk is ignored, although the frame information will still be processed. The add_site parameter is an integer added all Site values. This is a testing aid, allowing one to test PLACES upload using areas of the dev database that don't have old uploads in the way. It should NEVER be used in operations! Similarly, add_saved is added to all Saved Frame values. It also is merely a testing aid, and should NEVER be used in operations! The outStream parameter is a PrintStream object where all logging type messages will be written.
        Returns:
        0 on success
