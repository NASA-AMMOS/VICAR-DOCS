Package jpl.mipl.mdms.FileService.komodo.client.handlers.examples.slidingwindow

Package containing classes for MDMS Komodo Client Handlers plugin for LRU sliding window file tracking.

See: Description

  • Interface Summary 
    Interface Description
    Configuration
    Purpose: Configuration property names and some default values.
    QueryResultHandler
    Purpose: Interface for query ResultSet handler that performs an action for each result set entry.
    SlidingWindowFileDatabase
    Interface for Sliding Window database classes.
  • Class Summary 
    Class Description
    FileRecord
    Data structure represented a file record, including filename, full filetype, location, filesize and modification time.
    SlidingWindowDerbyFileDatabase  
    SlidingWindowFileDatabaseFactory
    Factory instantiates instances of SlidingWindowFileDatabase based on the value of a designated property 'slidingwindow.database.implementation', which can be set in a passed-in Properties map or via the System properties.
    SlidingWindowFileDatabaseTool
    Purpose: Utility tool that offers some simple services in maintaining the embedded database.
    SlidingWindowFileDatabaseToolDerbyOnly
    Purpose: Utility tool that offers some simple services in maintaining the embedded database.
    SlidingWindowFileHandler
    The primary purpose of this handler is to ensure a limit on disk space usage for subscribed files at remote locations.
    SlidingWindowH2FileDatabase
    Purpose: Initial implementation of the SlidingWindowFileDatabase using the H2 database.
    SystemInfo
    Purpose: System info is responsible for system level operations such as space metrics.
    SystemUtility
    Purpose: System utility is responsible for system level operations such as space metrics and file deletions.
    TestDbPerformance
    Purpose: Testing class that adds/updates entries into an SlidingWindowFileDatabase implementation.
