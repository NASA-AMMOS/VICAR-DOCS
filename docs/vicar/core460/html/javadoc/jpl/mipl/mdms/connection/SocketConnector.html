Class SocketConnector

  • java.lang.Object
    • jpl.mipl.mdms.connection.SocketConnector
  • All Implemented Interfaces:
    java.lang.Runnable, ServiceHandler

    public class SocketConnector
extends java.lang.Object
implements ServiceHandler
    Implementation of the Connector design pattern [POSA2] for initializing new Service Handler objects. This is a complete reimplemention of the original SocketConnector class due to New I/O support in JDK1.4 with multiplex non-blocking communication establishment. This implements the ServiceHandler interface that is required by the Reactor framework.

    Service handlers that are registered to this Connector will not be dispatched directy by the Reactor on connection establishment. Instead, the Connector will be dispatched and it will look up and notify the correct service handler object to initialize it self by invoking the service handler's open() method. After the initial connection notification, the service handler will be unregistered from the Connector's dispatch list. Any future events that the service handler 'wishes' to be notified is between the service handler and it's internal Reactor reference.
