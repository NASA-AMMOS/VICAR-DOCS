jpl.mipl.mica

Class ImageBag

  • java.lang.Object
    • jpl.mipl.mica.ImageBag
  • All Implemented Interfaces:
    java.beans.PropertyChangeListener, java.util.EventListener

    public class ImageBag
extends java.lang.Object
implements java.beans.PropertyChangeListener
    The ImageBag is responsible for managing the imagery components in the MICA system. The bag, at its most basic, is a container for holding all the ImageChips which comprise a mosaic. The global parameters of this mosaic are also stored here, as is the reference to the currently selected set of ImageChips. Further, the bag is also responsible for PropertyChangeEvent management.

    All ImageChips, once initialized, are added to the global bag. The Java Advanced Imaging (JAI) rendered operations chain that generates the mosaic image will be updated, then "Elements" property change events will be sent to all listeners. When removing chips, the JAI chain is again updated, then another "Elements" event will be fired. ImageChips can also be re-ordered so as to adjust their vertical depth-sorting (draw order). Light-weight methods are available which can be used to add or remove chips in batches without multiple events being fired.

    The ImageBag acts as the central message switcher in the MICA system. The bag registers itself with all of the chips to be a listener for any bound property changes they may have. The Display component as well as any tools register themselves with the ImageBag to receive any chip or bag bound property changes.

    The ImageBag also stores two objects which are global to the mosaic, namely the JPL Planetary Imaging Geometry (PIG) Projection and SurfaceModel. The bag is registered as an event listener on these objects as well. The preferred way of obtaining the mosaic's global properties is to obtain a reference to these PIG objects via the ImageBag: 

    Projection p=SVT.getImageBag().getProjection();

      • setOperation

        public void setOperation(java.lang.Object[] operation)
        Adds a JAI RenderedOp ParameterBlock to the ImageBag's ImageChain. The ImageChain will handle adding Sources to the ParameterBlock and call the JAI.create() method.

        The operation array block should comprise the following:

        • operation[0]="MyKeyName" - The key name used to retrieve or modify this operation later.
        • operation[1]="JAIOperation" - The string name for the JAI operation to add, delete, or modify.
        • operation[2]=new ParameterBlock() - The new or modified ParameterBlock used to configure the JAI operation.

        This will then cause the chip to fire an "Operation" message with the old and new operation arrays.

      • propertyChange

        public void propertyChange(java.beans.PropertyChangeEvent evt)
        This method implements the PropertyChangeListener for the ImageBag. It acts as the primary switching site for all messages in the MICA system. The ImageBag registers itself with all ImageChips so that all ImageBag and ImageChip bound property change events will be relayed from here. All tools or the global Display which need to listen for events register themselves to the ImageBag such that this method will relay all desired messages to them.
