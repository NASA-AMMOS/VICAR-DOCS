Class DataPlane

  • java.lang.Object
    • jpl.mipl.tlm.jedi2.DataPlane

    java.io.Serializable

    public class DataPlane
extends java.lang.Object
implements java.io.Serializable
    A model bean that that describes an array of data to be displayed in the JEDI client. The current properties used to do this are:
    • a band number, intended to be the band number from the original EDR data file, to associate with this DataPlane
    • a two dimensional array containing the data for the subject band in byte[line][samp] format
    • a Histogram. This can be either of the array itself or, more likely, a histogram of the original EDR data
    • the scaling factor used to scale the original DN values to the 8-bit representation used for transfer
    Both the client and the server will use this information to perform their function. The client can display a single DataPlane via multiple EDRCanvas views. Changing any of the properties contained in this class will require a new DataPlane for use by the client. I.e., while two EDRCanvas components could show the same data with different stretches, the same cannot be said for two EDRCanvas components wanting to use the same DataPlane with different DN scaling. The EDRLoader (via the server) will only know to load this DataPlane with a specific set of properties.
