    Created by an EDRReader

    This object should handle EDR sized data arrays fairly well. I can make no claims as to what will happen if you try to create an EDR instance that is extraordinarily large. Most likely, any limit will be set by the amount of memory you've configured for your JVM. Realistically, EDRs for use within JEDI should probably be under several MB in size as this class is intended to be passed around through serialization/RMI mechanisms.

    Anyway, this EDR class is designed to be a minimal definition of an EDR such that only the salient elements are serialized.

    Finally, it should be noted that my definition of an EDR, which is derived from the MPF examples I have, is that all the pertinent information is found in the VICAR Property label items. Though the Tasks elements are filled and used, it is the Property Map that is, seemingly, of most importance.

    This also enforces the assumption that all such keywords are unique within the VICAR label.
