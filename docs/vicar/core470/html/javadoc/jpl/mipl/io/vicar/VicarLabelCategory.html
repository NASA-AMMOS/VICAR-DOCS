jpl.mipl.io.vicar

Class VicarLabelCategory

  • java.lang.Object
    • jpl.mipl.io.vicar.VicarLabelCategory
  • All Implemented Interfaces:
    java.io.Serializable, java.lang.Cloneable

    public class VicarLabelCategory
extends java.lang.Object
implements java.lang.Cloneable, java.io.Serializable
    This class maintains a category of VICAR labels (i.e. a group of sets - all property sets or all history tasks). Unlike items within a set, the order of sets within a category *is* important (at least for history to indicate time order) so ordering is preserved. In addition, both history and property labels allow for multiple instances of the same name, so retrieval must be keyed off of both the name *and* the instance.

    Note that, while the instance number is stored within the VicarLabelSet object (to maintain self-descriptiveness), it is managed by this class to ensure that they count properly (starting at 1, incrementing by one). Specifically, this means that any modifications to the VicarLabelCategory structure could change the instance numbers of any or all sets (even ones for which the caller has already received a pointer).

    Even more importantly, instance numbers must not be modified directly by the caller, or undefined behavior will result. Use functions in this class to add or remove entries, instead.

    Per VICAR conventions, instances start counting at 1, not 0.

    Access to this object is not synchronized in any way. The caller must synchronize if necessary. ArrayLists are used instead of Vectors for efficiency.

      • fromXML

        public void fromXML(org.w3c.dom.Element category,
                    java.util.List errorList)
        Reads a VicarLabelCategory from an XML DOM node. Only sets that match the "flavor" of this category are read, so the flavor must be set in the category prior to calling this routine. For example, if the flavor is LABEL_PROPERTY, all of the <property> children will be read from the supplied Node, while <history> and <system> children will be ignored. See the VICAR label DTD. !!!!TBD: reference to this DTD!!!!

        Sets found in the XML are added to this category. This means that existing sets will be retained in the label (instance numbers will be set accordingly). If you want only the XML items, make sure the Category is empty before calling this function.

        If anything is wrong with the supplied node, error/warning messages will be appended to the supplied errorList. However, the parser keeps working as long as possible, despite errors. The practical upshot of this is that the errors can usually be ignored; the parser will do the best it can. However, the errors can be displayed to the user if desired. An exception will be thrown only if something unrecoverable happens.

        Parameters:
        category - The Element from which to extract the sets.
        errorList - A list of Strings to which any parsing errors will be appended. Can be null, in which case no errors are returned.
        Throws:
        org.w3c.dom.DOMException - if something unrecoverable happens. Improperly formatted items or violations of the DTD do not necessarily result in exceptions; parsing continues as long as possible and problems are appended to the errorList.
        See Also:
        VicarLabelSet.fromXML(Element, List)

      • toLabelString

        public VicarLabelCategory.ItemIndex toLabelString(java.lang.StringBuffer buf,
                                                  int max_length,
                                                  VicarLabelCategory.ItemIndex start_pos)
        Returns in the supplied StringBuffer the part of the category from start_pos to the end as a valid VICAR label string, ready to be put in a file. If the max_length argument is non-zero, it represents the maximum size the string can become (normally used to make sure the label fits in the space at the front of a file). This includes any data already in buf when the function is called. A 0 means no maximum.

        Note that the caller is responsible for putting in LBLSIZE= at the beginning of each buffer's label segment.

        Parameters:
        start_pos - A VicarLabelCategory.ItemIndex structure containing the set and item indices of the first item to be included in the string. If null, the label is written from the beginning.
        Returns:
        A VicarLabelCategory.ItemIndex structure containing the set and item indices of the first item not included in the string. This should be passed into start_pos on the next call. Returns null if the entire label was written.

      • recalculateInstances

        public void recalculateInstances()
        Recalculates instance numbers for all sets. This is normally not needed, as instance numbers are maintained automatically. However, if the caller does something really nasty to a Set, such as changing its name, without going through the Category, the instances could get confused. It is up to the user to call this routine in that case. Note that this is implemented very inefficiently, because it should not be needed.
