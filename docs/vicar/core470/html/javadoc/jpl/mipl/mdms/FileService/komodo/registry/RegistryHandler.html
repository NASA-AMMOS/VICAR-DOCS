    public interface RegistryHandler
    Interface for registry handler. This defines the required interface for the komodo core to interact with the registry implementation. The registry interface requires all method calls to include a handler to handle the results returned from the registry. If errors occurs, the handler is expected to handle them.

      • handleResult

        void handleResult(java.util.Hashtable result)
        Method to handle the result returned from the registry. The results are packaged in a hashtable. The handler implementation is expected to know the keywords to use to retrieve the result objects. This method is invoked each time a result record is returned. That is, if the registry is implemented using RDBMS, then each row returned will trigger this method to be invoked.
        Parameters:
        result - a hashtable that maps the return result set into keyword + value.

      • hasError

        boolean hasError()
        Accessor method to return true if an error was encountered. Since this interface is intented to encapsulate the exceptions that might be thrown by the registry implementation, so that the calling object won't have to hardcode and catch registry-specific exception type. One way to inform the calling object that an error was encountered is by capaturing it into this object and set an error flag.
        Returns:
        true if an error was encountered.
