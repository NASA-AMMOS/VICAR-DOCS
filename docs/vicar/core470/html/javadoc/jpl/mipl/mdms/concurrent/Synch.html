      • acquire

        void acquire()
      throws java.lang.InterruptedException
        This method blocks (may be forever) until it has successfully obtain a lock. The only other way for this method to have early exit is by an interruption. If the caller receives an exception during this method call, it means the attempt to obtain the lock failed.
        Throws:
        java.lang.InterruptedException - when thread is interrupted

      • acquire

        boolean acquire(long msec)
         throws java.lang.InterruptedException
        Timed acquire. This method tries to obtain a lock within the specified milliseconds time. The time variable does not guarantee the method will retun at the exact time, due to the architectural limitation of the Java language. In addition, the internal timers in Java does not stop during garbage collection, so timeout can occur because of GC intervended.
        Parameters:
        msec - the number of milliseconds to wait. If a value of zero or negative is assigned, then the method will not wait.
        Returns:
        true if acquire.
        Throws:
        java.lang.InterruptedException - when thread is interrupted

      • release

        This method releases the acquired lock to allow other thread(s) to have access to the shared resource.
