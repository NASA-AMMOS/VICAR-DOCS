    Pliers - PLaces Information Extraction for Rover and Site files

    This class creates the "database" of XML files needed by RSVP. It consists of a site file containing all the Site offsets, and a rover file for each Site containing the Rover frames found within the Site.

    Although each of these can be done with a simple query, we need a program to perform the query on Sites, then interpret the Site file so we know which Rover files to create.

      • getPwdFromNetrc

        public static void getPwdFromNetrc(java.lang.StringBuilder passwd,
                                   java.lang.StringBuilder userid,
                                   java.lang.String url_string)
        Attempt to read the .netrc file to get the password token. If it fails for any reason, quietly return null. This is only likely to work on Unix-based machines... on others, the .netrc file will not be found and it will return null. If the userid is null, it'll return the first entry matching the host. Otherwise it'll match the userid. Both the pwd and userid's are StringBuilder objects so we can easily return two strings. They should both be empty when called (or userid set to the username).

      • doAbbreviate

        protected static void doAbbreviate(org.w3c.dom.Document doc,
                                   boolean isSite)
        Abbreviate the XML for ease of human readability by eliminating the following:
        • solution_metadata element
        • origination element
        • add_date, immutable, xmlns attributes from solution
        Additionally, if isSite is true, the orientation tag is removed.

        This should ONLY be used for ease of reading by humans - NOT for machine readable output files!

        Not really needed for bulk trans output; this does nothing for those.
