Class MarsPerstoreImpl

  • java.lang.Object
    • jpl.mipl.mars.pgs.perstore.MarsPerstoreImpl
  • All Implemented Interfaces:
    java.rmi.Remote, ServerBase_IF, FileLock_IF, Groups_IF, Instrument_IF, IntrPerStore_IF, PersistentStore_IF, ProcessInstance_IF, ServerConfiguration_IF, SFConfiguration_IF, UserInformation_IF

    public class MarsPerstoreImpl
extends java.lang.Object
implements IntrPerStore_IF
    This is an implementation of the Perstore interface for MSL. MSL currently does not use any database, in fact the only thing that this class will have to handle will be the configuration for: a. controller b. service factories It will get this info from the MSL Configuration file MSLServerConfig.xml The methods that do any work deal with getting controller and sf configuration and creating a dummy PGSSession
    Author:
    Cecilia Cheng (csc@mipl.jpl.nasa.gov);
