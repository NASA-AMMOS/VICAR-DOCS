Class EDRLoader

  • java.lang.Object
    • jpl.mipl.tlm.jedi.EDRLoader
  • All Implemented Interfaces:
    EDRReader

    public class EDRLoader
extends java.lang.Object
implements EDRReader
    This class determines what type of file the target EDR is (VICAR, ISIS, FITS, ...) and loads the data accordingly. Objects that use EDRLoader for this purpose do not need to concern themselves with the filetype, though they do need to check for a FormatNotSupportedException. (which I haven't implemented yet :-/ )

      • setDNScaling

        public void setDNScaling(double choice)
        Description copied from interface: EDRReader
        Determines how the data should be scaled to 8-bit. Normally, the Reader will scale the data based upon discrete divisors. I.e.:
        max DNdivisor
        255 1
        511 2
        1023 4
        2047 8
        4095 16
        8191 32
        16383 64
        32767 128
        65535 256
        etc...
        This is designed to keep the histogram consistent with the dynamic range that the sensor is capable of. However, one may wish to have the image DNs scaled such that the maximum DN of the data is used to compute the scaling. This can cause the histogram to be much more filled. In this case, the histogram wouldn't be as correct, but likely more appealing.
        Specified by:
        setDNScaling in interface EDRReader

        public void setScalePolicy(boolean choice)
        Description copied from interface: EDRReader
        If the EDR is smaller than the Personality requires, this allows for the Reader to know what it should do about scaling. The reason this and the setDesiredSize methods exist is to alleviate the necessary network bandwidth to support any given Personlity. We simply shove the least amount necessary across the wire.
