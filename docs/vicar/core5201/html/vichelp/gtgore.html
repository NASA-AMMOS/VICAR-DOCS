Help for GTGORE

PURPOSE
     GTGORE is a VICAR applications program which calculates gore
     adjustments for GTWARPXD (see its subroutine GTWRPSUB).  It
     is not intended for general use.
	   
CALL
     gtgore (GRID_A,GRID_B) ACOLS=(C1,C2,C3,C4) BCOLS=(C1,C2,C3,C4)             SLVAR=INCSL SSVAR=INCSS NLVAR=INCNL NSVAR=INCNS
  WHERE:
     GRID_A         is the first grid in GEOMV format.
     GRID_B         is the second grid in GEOMV format.
     ACOLS          are the columns used in the first ibis file.
     BCOLS          are the columns used in the second ibis file.
     SLVAR          is a TAE TCL variable to output the SL increment
     SSVAR          is a TAE TCL variable to output the SS increment
     NLVAR          is a TAE TCL variable to output the NL increment
     NSVAR          is a TAE TCL variable to output the NS increment
     
OPERATION

GTGORE operates like gridcomp except that.

PERFORMANCE

faster than gridcomp

Restrictions
------------

Only a 40 x 40 grid can be input as the second file.  The first file must
be in GEOMV input format.  Both input files are IBIS files.

PARAMETERS:

INP

 IBIS tables containing the two grids to be composed.

ACOLS

 Columns in the first input

BCOLS

 Columns in the second input

SLVAR

 TAE TCL variable to receive the integer value of the adjustment of the image SL

NLVAR

 TAE TCL variable to receive the integer value of the adjustment of the image NL

SSVAR

 TAE TCL variable to receive the integer value of the adjustment of the image SS

NSVAR

 TAE TCL variable to receive the integer value of the adjustment of the image NS
