    public class ToolPalette
extends javax.swing.JPanel
implements java.awt.event.ComponentListener, java.awt.event.WindowListener
    The ToolPalette is a container of tools. It is a Swing component which manages Java archives (jar files) of java beans to be used as MICA tools. The tools are listed in the visual component and can be activated by clicking on their icon or name. Each archive contains a tool set and each tool set is presented as a branch of a collapsable tree. The ToolPalette also manages a preferences file for tool options. The file, by default, is located in the user's home directory (~/.jpl.mipl.mica on Unix-like systems) and consists of name-value pairs for the properties the ToolPalette and various tools have stored there.

      • loadToolArchive

        public void loadToolArchive(java.lang.String theJar)
        Loads a tool set from a named path. A ClassLoader is created specifically for this path. Note that we call findResource() so that only tools in this specific path (jar) will be loaded. We don't want to re-load everything in the system path, which loadToolArchive(ClassLoader) does.

        In addition, tools loaded via this method will be marked with "LoadToolset" in the preferences file, so they will be automatically re-loaded next time MICA is started. Note that there is no facility for auto-unloading of things that are in classpath; that has to be done manually.

        Parameters:
        String - File name of the jar containing the tool set, or pathname of the directory (ending in '/') containing the classes and Spi file.
        See Also:
        for a description of the tool file and its contents.

      • loadToolArchive

        public void loadToolArchive(java.lang.ClassLoader loader)
        Loads all tool sets that are available in the given class loader. Tool sets are searched for in the file:

        META-INF/services/jpl.mipl.mica.ToolPalette

        which can be in every jar file (and in each directory in classpath, if not using jars). Entries in this file contain "tool " followed by the fully qualified class name of the tool bean. For example: 

         tool jpl.mipl.mica.tools.Order
 tool jpl.mipl.mica.tools.Pointing
 tool jpl.mipl.mica.tools.Selection

        The bean name and icon (from the bean's BeanInfo) are then used to add an entry into a subfolder in the ToolPalette's component window. An entry is also made in the ToolPalette preferences indicating the name and archive file name.

        Parameters:
        ClassLoader - The class loader to use, or null to use the system one.

      • processResourceURL

        protected void processResourceURL(java.net.URL url,
                                  java.lang.ClassLoader loader,
                                  boolean autoload)
        Process one URL as loaded from a services file. The URL is a file containing tool definitions. Entries in this file contain "tool " followed by the fully qualified class name of the tool bean. For example: 
         tool jpl.mipl.mica.tools.Order
 tool jpl.mipl.mica.tools.Pointing
 tool jpl.mipl.mica.tools.Selection

        The bean name and icon (from the bean's BeanInfo) are then used to add an entry into a subfolder in the ToolPalette's component window. An entry is also made in the ToolPalette preferences indicating the name and archive file name.

        Other types of entries (besides "tool ") may be added in the future.

        Note: the parser is not particularly smart; a single space must separate "tool" from the pathname. However, comments (# in column 1) are allowed.

        Parameters:
        url - URL of the tool file, usually from META-INF/services.
        loader - The class loader to use for the file.
        autoload - True if the tool should be marked with "LoadToolset" in the preferences file, so it will be loaded automatically on startup.
        See Also:
        loadToolArchive(ClassLoader), loadToolArchive(String)

      • setPrefProperty

        public final void setPrefProperty(java.lang.String prefName,
                                  java.lang.Object prefValue)
        This method adds (or modifies) a preferences property with the name and value pair given. Preferences names have a dot-separated hierarchy that should be adhered to. 
        • Toolset.[tool set name]
          - a tool set name and archive file name. 
        • [tool name].XMin
          - a tool's preferred screen X location. 
        • [tool name].YMin
          - a tool's preferred screen Y location. 
        • [tool name].Width
          - a tool's preferred width. 
        • [tool name].Height
          - a tool's preferred height.
        Other properties can be set by the tools, but should be prefaced by the tool's name.

        The ToolPalette attempts to save the preferences to disk when it is dispose()'d of.

      • getToolBounds

        protected final java.awt.Rectangle getToolBounds(java.lang.String name,
                                                 java.awt.Dimension size)
        This method tries to intelligently position tools on the screen when they are opened by the ToolPalette. First, if any of the four bounds preferences are set (X and Y locations, width and height), they are used. Any unset bounds are set using defaults. Second, if no preferences are set at all, the method makeToolBounds() is called to try to fit the tool on screen without overlapping an already open tool.
        Parameters:
        String - Tool name, for checking the preferences.
        Dimension - Tool preferred width and height.

      • makeToolBounds

        protected java.awt.Rectangle makeToolBounds(java.awt.Dimension d)
        Utility method to try to find an open space on the screen large enough to fit a component of the given Dimension. It scans down the right edge of the main Editor window looking for a large enough gap in the open tools' windows. It gets the screen size from AWT and will return a position just off the top right of the Editor if no open space meets its selection criteria.

        NOTE: This method may be overridden to provide better placement intelligence.

        Parameters:
        Dimension - Preferred tool width and height, or null if none are available.
