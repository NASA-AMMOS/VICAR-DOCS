    public class Zoom
extends javax.swing.JPanel
implements java.beans.PropertyChangeListener
    A Zoom tool is used to adjust the view location and pixel resolution of an output mosaic. The tool is designed to work with a valid jpl.mipl.mica.Display and a valid jpl.mipl.mica.projection. It includes graphical widgets for a user to adjust the location of the Display's viewport on its JadeDisplay. It includes widgets to set the Projection's scale so as to adjust the output mosaic's pixel resolution (and the mosiac width and height that depend on the resolution).

    The tool includes a scaled down representation of the entire mosaic. This is assembled using Java2D and Swing rather than JAI. The purpose of this is to ensure that this tool does not appreciably slow down the MICA system. It renders chip images individually and does not force ImageChips to be rendered if they are not already rendered. This behavior is different from simply scaling down the full mosaic because this would force all tiles of that full mosaic to be computed thereby forcing all ImageChip images to be computed. This would eliminate the advantages of the JAI deferred rendering paradigm. Instead, a Zoom tool will render the border of the chips, which has been computed regardless. This results in an approximation of the full mosaic in the tool's graphical display.

    A Zoom tool also allows the Projection scale to be adjusted. While the mosaic zoom factor and the Projection scale are separate concepts, the Display's JadeDisplay treats them as identical. This is done because other aspects of the MICA system operate at maximum efficiency only when the mosaic (and its component ImageChips) is rendered at the same resolution as the output. Because rerendering the mosaic can be computationally expensive, an adjustment delay is used. When any scale adjustment is made, the user is given 1.5 seconds to make additional adjustments before the final scale is commited and a new mosaic rendered.

      • setProjectionScale

        protected void setProjectionScale(double s)
        Defer an attempt to set the global Projection's scale. If, after 1.5 seconds no further action is taken to stop or reset this deferred attempt, the scale will be reset. Further, all ImageChip reprojectedImage buffers will be flushed using ImageChip.doReprojection(). The code attempts to keep the viewport in the same apparent location despite the changing coordinate scales.
