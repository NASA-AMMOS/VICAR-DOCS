    public class StereoJFrame
extends javax.swing.JFrame
    Top-level container that is capable of displaying swing components in stereo. Intended to be used in place of Swing's JFrame.

    This is an extension of JFrame based on Agile2D's AgileJFrame. Major change is an addition of support for stereo display. Also overrides JFrame's getGraphics() to return custom Graphics2D object(if available) that supports drawing in stereo.

    This implementation of JFrame uses a "pluggable renderer" mechanism to manage rendering the contents of the frame.

    Each StereoJFrame has a PluggableRenderer associated with it. The PluggableRenderer is responsible for rendering performed in the frame. It also provides additional methods for controlling the renderer at a high level.

    Currently, the only implementation that is stereo-capable is JOGL OpenGL renderer.

    Normal JPanel or other Swing containers can be used within this component; they will be rendered to both eyes. At the point where stereo UI is desired, use a StereoJPanel.

    Usage:

    StereoJFrame stereoJFrame = new StereoJFrame();
    // Note that StereoJFrame can manage any component as content pane's
    // child but to display components in stereo as "left" or "right", user has
    // to use StereoJPanel as content pane's child
    StereoJPanel stereoJPanel = new StereoJPanel(); stereoJPanel.add(swing_component_1, "left"); stereoJPanel.add(swing_component_2, "right"); stereoJFrame.getContentsPane().add(stereoJPanel);
    //see Jadis for description of supported stereo modes
    stereoJFrame.setStereoMode(Jadis.STEREO_ANAGLYPH);
    stereoJFrame.setVisible(true);

    When StereoJPanel is used, for certain Swing operations like setting cursor, it's not advisable to do it on "left" or "right" component. Instead do it on it's parent, StereoJPanel and "left" and "right" children will inherit this attribute. Otherwise, if user, for example would setCursor on "right" component, then "left" component wouldn't "know" about it and use a default cursor or whatever cursor was set on it. Then it will be up to the Swing to determine which component, "left" or "right" owns a given mouse event, and that decision will determine which cursor to use. So to avoid the risk of being "ignored" by Swing, set attributes that has to be shared by "left" "right" components on it's parent StereoJPanel If, by some reason, setting cursor(and other resources) on "left" and "right" parent, StereoJPanel is not feasable, make sure set/unset cursor(and other resources) on BOTH "left" and "right" component.

    See Also:
    StereoJPanel, Jadis, PluggableRenderer, Serialized Form
