      • compareByGeometryString

        protected int compareByGeometryString(java.lang.String geoStr1,
                                      java.lang.String geoStr2,
                                      boolean flip)
        When it comes to sorting, the earlier a product appears in the product pipeline, the 'earlier' it should be in the sorting. Bob Deen wants *later* compression (which is represented by geometry string...) types to appear earlier (so that it appears in the EDR group instead of the one with an underscore). As such, we have this method which accepts a boolean flag to indicate we wish to flip the ordering.
        Parameters:
        geoStr1 - Geometry string for the first product
        geoStr2 - Geometry string for the second product
        flip - If true, then treat 'later' products as earlier.
        Returns:
        sort value
