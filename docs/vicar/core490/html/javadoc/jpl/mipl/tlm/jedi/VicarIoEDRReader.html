Class VicarIoEDRReader

  • java.lang.Object
    • jpl.mipl.tlm.jedi.VicarIoEDRReader
  • All Implemented Interfaces:
    EDRReader

    public class VicarIoEDRReader
extends java.lang.Object
implements EDRReader
    Creates an EDR object from a VICAR EDR on disk using JAI and Bob Deen's/Steve Levoe's jpl.mipl.io.* packages. Basically, to use this object, you simply instantiate it with a VICAR, PDS, or dual-labelled PDS/VICAR file as input, stand back, and hope all goes well.

    This object should handle EDR sized images fairly well. I can make no claims as to what will happen if you feed it a huge dataset.

    Lastly, this class was intended to offer a way for an EDR to be organized into a Java Object such that it could then be passed around through serialization/RMI mechanisms.

      • setDNScaling

        public void setDNScaling(double choice)
        Description copied from interface: EDRReader
        Determines how the data should be scaled to 8-bit. Normally, the Reader will scale the data based upon discrete divisors. I.e.:
        max DNdivisor
        255 1
        511 2
        1023 4
        2047 8
        4095 16
        8191 32
        16383 64
        32767 128
        65535 256
        etc...
        This is designed to keep the histogram consistent with the dynamic range that the sensor is capable of. However, one may wish to have the image DNs scaled such that the maximum DN of the data is used to compute the scaling. This can cause the histogram to be much more filled. In this case, the histogram wouldn't be as correct, but likely more appealing.
