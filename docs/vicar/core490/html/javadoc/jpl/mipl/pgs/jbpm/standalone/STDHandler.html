Class STDHandler

  • java.lang.Object
    • java.util.logging.Handler
      • jpl.mipl.pgs.jbpm.standalone.STDHandler
  • All Implemented Interfaces:
    java.io.Externalizable, java.io.Serializable

    public class STDHandler
extends java.util.logging.Handler
implements java.io.Externalizable
    This is a simple log handler to be used with the standalone PGS execution engine. This handler acts like a rolling file appender of the Log4J logging framework; when the log file reaches the specified size, it is renamed and a new log file with the original name is created. Although the publish method is synchronized, there is no guarantee that logging may not collide. For instance, using same instance of handler in multiple pipeline may create message collusion.
