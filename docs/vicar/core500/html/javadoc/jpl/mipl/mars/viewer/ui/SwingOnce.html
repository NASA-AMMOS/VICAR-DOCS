jpl.mipl.mars.viewer.ui

Class SwingOnce

  • java.lang.Object
    • jpl.mipl.mars.viewer.ui.SwingOnce

  • public class SwingOnce
extends java.lang.Object
    Purpose: A SwingOnce coalesces invocations of a (series of) Runnable(s) in the Swing thread such that only the last in the series is actually run. If a Runnable is scheduled while one is already pending, the first is canceled (even if the two Runnables are different). Otherwise, the SwingOnce schedules the Runnable for later execution in the Swing thread, and the Runnable will be run iff no other Runnable is scheduled in the same SwingOnce first. This is useful when you only care that the last Runnable in a series is actually run (as you might when the Runnable's job is to set a cursor position) or when all Runnables in a given series are the same and all that matters is that one of them is run. Note that none of the methods is synchronized, and no locking is performed internally. This is because, in all the cases where I'm currently using it, a given SwingOnce is only used from a single thread. Props to Scott Maxwell for this.
