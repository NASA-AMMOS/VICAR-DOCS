      • registerMission

        void registerMission(Project missionName,
                     boolean regFlag)
              throws BadServiceException
        Registers to recieve events for a given mission name. This may be combined with other mission or mission instrument registrations. Also, you may register for several mission names at once. If you specify true for the regFlag, the registration string will immediately be sent to the notification server. If you specify false then the registration string will only be updated locally. This allows several changes to the subscription to be made locally with only the last changed updating the notification server.
        Parameters:
        missionName - The name of the mission.
        regFlag - Indicates if the new registration string should be updated in the notification server.
        Throws:
        BadServiceException - Thrown for CORBA exceptions.

      • logMessageOn

        void logMessageOn(boolean regFlag)
           throws BadServiceException
        Turns LogMessage events on This will allow you to recieve LogMessage events. By default, you will recieve all events (StreamStarted, ProductAvailable and StreamStopped). Pass a zero to regFlag if you want the function to product the filter expression string, but not actually update it in the Notification server. This is useful if you're doing several subscription changes in a row and only want to update the server on the last one.
        Parameters:
        regFlag - Submit true if you wish to update the registration on the server, false if not.
        Throws:
        BadServiceException - This exception is thrown when there is some CORBA problem.
