    A JAI operation to call the Planetary Image Geometry (PIG) library to transform a source image.

    This operator takes a single ImageChip and reprojects its pixels into an output coordinate space defined by a global SurfaceModel and Projection. The source imagery is a raster of pixels, such as from a CCD camera, as viewed from a measured camera location and orientation. The resulting image is the modified imagery, as it would be seen from the output camera location.

    The algorithm is a raytracer. First, the approximate bounds of the reprojected image in output coordinates are determined. The actual border of the image is traced precisely, then the bounding box of this border is taken. The pixels inside this bounding box are then raytraced at the given output resolution. The raytrace process is that of the JPL raytracer known as marsmap.

    For more information on marsmap, the PIG library, and other related software, see MIPS.

    The marsmap algorithm calls into the PIG library to identify the source pixel (if any) that should be rendered at the output location. This involves a transformation from pixel coordinate space into a three-dimensional look vector in that direction, using the controlling Projection object. This vector is intersected with the PIG SurfaceModel to determine the point on the surface that would be the subject of a source image. A dot-product is then performed between the look vector and the orientation vector for the source image camera. If the look vector is within the camera's field of view, then the next step is performed. The surface point is converted from three-dimensional cartesian coordinates in output space into two-dimensional raster coordinates in source image camera space. Finally, if this two-dimensional coordinate corresponds to a pixel in the source image file, this coordinate is returned.

    This operator also provides funcitonality for lower-resolution, quicker computation. The pixel skip and border options can make a reprojection return a partial pixel-mapping. The raytracing of output pixels proceeds at a finite rate which is not sufficiently fast to provide real-time feedback to the user. Techniques may be enabled which provide near-real-time graphical feedback by limiting the computation to a set number of pixels.

    The arguments passed in via the ParameterBlock are:

    • ImageChip: The chip to raytrace.
    • Boolean: Use the cache on the ImageChip to buffer the imagery?
    • Integer: Pixel skipping value, in pixels. A value of 1 means all pixels will be computed. A value of 2 means that every other pixel will be computed, and so on.
    • String: Focus computing effort on certain pixels, either a wipe or a spot. Unimplemented.
    • Integer: Focus step size, in pixels.
    • Boolean: Should the output image have alpha transparency?
    • Integer: Transparency type, if using alpha (see Transparency interface).
    The optimal configuration, by taking advantage of native code libraries, uses:
    • a single banded source image,
    • a BITMASK alpha transparency, and
    • source data type unsigned-short.
