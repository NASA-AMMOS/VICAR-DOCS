Interface VicarIOBase

  • All Known Subinterfaces:
    VicarInput, VicarOutput
    All Known Implementing Classes:
    ISISInputFile, MgnFbidrInputFile, MgnResampInputFile, PDS4InputFile, PDSInputFile, PDSNativeInputFile, PDSOutputFile, VicarInputFile, VicarInputImage, VicarOutputFile, VicarOutputImage

    public interface VicarIOBase
    This interface includes everything that's common between the VicarInput and VicarOutput interfaces, such as file opening and label access. Since some classes are likely to implement both interfaces, it is cleaner to put the common things here rather than duplicate them in each interface.

    See Also:

      • getVicarLabel

        VicarLabel getVicarLabel()
                  throws java.io.IOException
        Retrieves a deep copy of the VicarLabel object associated with this file. In most cases, the VicarLabel object must be created from the header of the file. This routine should not be used to retrieve the system label bean; use getSystemLabel() instead.

        Note that requesting the label via this routine will cause the EOL labels to be read if needed. On a sequential-only stream, this could get you into trouble and thus this should be done only after the image data has been read. If random access is possible but not easy, this can be done before reading the data but you could take a significant performance hit. If it is easy, then this can be called any time.

        For existing images, e.g. a file opened for read or update, this routine returns the label of that image. For images implementing VicarOutput, however, the situation is a little more complex. This routine will return the first item from the following that is set:

        • Label set via setVicarLabel
        • Label from file itself if the file already exists
        • Primary input's label
        • Empty (newly constructed) label

        Throws:
        java.io.IOException - if an EOL read is required, and fails. Will not occur for random-easy streams.
        See Also:
        getSystemLabel()

      • getSystemLabel

        SystemLabel getSystemLabel()
        Retrieves a deep copy of the SystemLabel object associated with this image. This should not be confused with VicarLabel.getSystem(), which returns a VicarLabelSet rather than a SystemLabel object. The SystemLabel object returned by this function need not contain the same information as the System part of the VicarLabel object (e.g. a synthetic image made up of multiple files).

        For existing images, e.g. a file opened for read or update, this routine returns the label of that image. For images implementing VicarOutput, however, the situation is a little more complex. This routine will return the first item from the following that is set:

        • Label set via setSystemLabel
        • Label from file itself if the file already exists
        • Primary input's label
        • Default (newly constructed) label

      • createSampleModel

        java.awt.image.SampleModel createSampleModel()
        Creates a SampleModel that is most compatible with the image. The algorithm is subject to change, so you must not depend on it, but currently it is:
        • If the file is double, float, or complex, create ComponentSampleModelJAI. (complex creates twice as many bands)
        • Else, if the organization is BIP, create PixelInterleavedSampleModel.
        • Else, if the organization is BIL, create ComponentSampleModel.
        • Else, create BandedSampleModel (BSQ).
        The size of the sample model is the default VICAR tile size, which is currently 100 lines by 2048 samples (or smaller, if the file size is smaller). If the file is sequential-only or random-hard, the default tile width is set to the image width so we don't have to back up.

        This routine is mainly intended for input images; on output you usually already have a SampleModel. However, if you don't care what SM you use, this routine will return one that fits the output file.

      • createSampleModel

        java.awt.image.SampleModel createSampleModel(int tileWidth,
                                             int tileHeight)
        Creates a SampleModel that is most compatible with the image. The size of the sample model is specified by the parameters. If a parameter is specified as 0, the size of the file is used. Note that the size of the SM is not limited by the size of the file; you can create an SM as big as you want with this method.

        Follow the See Also link for algorithm details.
