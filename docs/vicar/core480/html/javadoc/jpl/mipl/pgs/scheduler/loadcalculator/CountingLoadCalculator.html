Class CountingLoadCalculator

  • java.lang.Object
    • jpl.mipl.pgs.scheduler.loadcalculator.CountingLoadCalculator
  • All Implemented Interfaces:
    SFLoadCalculator_IF

    public class CountingLoadCalculator
extends java.lang.Object
implements SFLoadCalculator_IF
    A simple load calculator that keeps track of how may pipelines of what kind it is currently running and based on the max for each of those pipeline, returns the load. In other words, if (currently_running!=max) loadfactor = currently_running/max else loadfactor=CANNOT_HANDLE_THIS_REQUEST

    rrp {rrp@jpl.nasa.gov}

        request - The request for which the load factory is requested.
        Returns:
        A float value between 0.0 and 1.0 indicating the current load of the factory. 1.0 means the factory is running at it's max capacity. Return CANNOT_HANDLE_THIS_REQUEST to indicate that the ServiceFactory this load calculator is associated with cannot run the given request now.
