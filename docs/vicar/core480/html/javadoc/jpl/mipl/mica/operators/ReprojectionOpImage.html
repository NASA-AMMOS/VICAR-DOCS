    public class ReprojectionOpImage
extends javax.media.jai.OpImage
    A JAI operation to call the Planetary Image Geometry (PIG) library to transform a source image. A ReprojectionOpImage should not be instantiated directly, but through the JAI operation registry.

    ReprojectionOpImage performs the JAI operation corresponding to the JPL VICAR program "marsmap" using the Planetary Image Geometry (PIG) library. See the ReprojectionDescriptor for more detail.

    Once rendering has been performed, a flag property named REPROJECTED will be set to Boolean.TRUE.

    For more information on marsmap, the PIG library, and other related software, see MIPS.

      • marsmap

        protected PigPoint2D marsmap(PigPoint2D output)
        Perform the marsmap.cc raytracing algorithm on the source images.

        The algorithm calls into the JPL PIG library to map the source pixel coordinate that corresponds to an output pixel location. If there is no corresponding input pixel in the image, a value of no_input is returned which will translate into a zero pixel value.

        The marsmap algorithm calls into the PIG library to identify the source pixel (if any) that should be rendered at the output location. This involves a transformation from pixel coordinate space into a three-dimensional look vector in that direction, using the controlling Projection object. This vector is intersected with the PIG SurfaceModel to determine the point on the surface that would be the subject of a source image. A dot-product is then performed between the look vector and the orientation vector for the source image camera. If the look vector is within the camera's field of view, then the next step is performed. The surface point is converted from three-dimensional cartesian coordinates in output space into two-dimensional raster coordinates in source image camera space. Finally, if this two-dimensional coordinate corresponds to a pixel in the source image file, this coordinate is returned.

        Parameters:
        output - Pixel coordinate to raytrace.
        Returns:
        Source pixel coordinate (or no_input).

      • getBorder

        public java.awt.Shape getBorder(int frequency)
        This method creates a java.awt.geom.GeneralPath which traces the edge of the ImageChip property. This path steps along each edge of the source image, starting from the upper left corner and proceeds counter-clockwise. The frequency of interim points per edge can be specified so that the border can be either very detailed or computed quickly.

        As the algorithm proceeds counterclockwise, it may strike the "zero meridian" boundary. The output mosaic image may wrap around to connect to another edge. In such a case, the algorithm determines where the path will re-emarge on the output Raster. The GeneralPath may need to be comprised of two distinct regions of the Raster. The interior of the border can be determined using the GeneralPath.WIND_NON_ZERO winding rule and any of the GeneralPath contains() or intersects() methods.

        This method calls inverseMarsmap() on each source edge pixel to determine the output coordinate mapping.

        Parameters:
        frequency - Number of points per edge to compute.
        Returns:
        The Java2D GeneralPath which traces the edge counterclockwise.
