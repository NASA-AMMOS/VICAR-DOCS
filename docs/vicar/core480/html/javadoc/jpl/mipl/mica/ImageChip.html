jpl.mipl.mica

Class ImageChip

  • java.lang.Object
    • jpl.mipl.mica.ImageChip

  • public class ImageChip
extends java.lang.Object
    The ImageChip is the basic unit of a mosaic. A set of such ImageChips, when held in an ImageBag, can be assembled and manipulated to form a mosaic.

    The ImageChip is largely a wrapper for a number of more basic objects that are all related to the same image. There are three Planetary Imageing Geometry (PIG) objects and three Java Advanced Imaging (JAI) objects inside of an ImageChip and these are acessible via the ImageChip's methods.

    The PIG objects are a PointingModel, a CameraModel, and a FileModel. Used together, these objects access and correct the environmental and equipment conditions under which an image was taken. The most important parameters stored on these objects are the camera type, location, and orientation at the time the image was taken.

    The three JAI objects are the base image, the reprojected image and the image chain. The base image is the raw imagery loaded from a file via a codec. It may also have been further manipulated (see the conditionImagery() method). The reprojected image is a ReprojectedOpImage that provides a view of the image data from a location other than the given camera location. The image chain is a set of JAI operations which reside on top of the reprojected image to add annotations or otherwise modify the image.

    NOTE: To load in an ImageChip, it is preferable to use a jpl.mipl.mica.util.ImageLoader. If the file is known, however, a chip may be directly instantiated. Before it can be used, however, the file name must be set.

      • originalPointingParms

        public double[] originalPointingParms
        Each chip has an original array of pointing parameters, which were determined by the instrumentation on the camera and its positioning hardware. Any changes made to the pointing parameters on this image chip are considered "corrections" to these direct measurements.

        These original values are used when applying "nav file" pointing corrections.

      • getImage

        public java.awt.image.RenderedImage getImage()
        Returns a reference to the JAI rendered operator used to generate the complete image of the reprojected image chip in the current global projection. This image includes any modifications that tools have requested to the image by inserting operators into the image chain.

        NOTE: The pixels in this image have not necessarily been computed. Because of the JAI deferred rendering scheme, pixel rendering occurs only as needed and the results are buffered only if system resources are adequate.

      • getBaseImage

        public javax.media.jai.PlanarImage getBaseImage()
        Returns a reference to the JAI rendered PlanarImage that is the base image. The base image is the raw pixel data, as loaded from the source data file but with certain necessary modifications. When first loaded, the image was inspected to ensure that it had a Java2D ColorModel and SampleModel compatible with the MICA rendering system. If it did not, it was converted to unsigned short sized pixels and all pixel values were stretched to fit this new pixel bit width.

        NOTE: The pixels in this image have not necessarily been computed. Because of the JAI deferred rendering scheme, pixel rendering occurs only as needed and the results are buffered only if system resources are adequate.

      • getReprojectedImage

        public javax.media.jai.PlanarImage getReprojectedImage()
        Returns a reference to the JAI rendered PlanarImage that is the reprojected image. This image is cached here on the ImageChip to speed performance at the expense of higher memory consumption. To test if a reprojected image is being cached, check the returned value first to see if it is null. If it is not, then an operator has been cached for the current ImageChip state. To then determine if the pixels have yet been computed, check to see if the flag property named "REPROJECTED" has been set to Boolean.TRUE.

        NOTE: The pixels in this image have not necessarily been computed. Because of the JAI deferred rendering scheme, pixel rendering occurs only as needed and the results are buffered only if system resources are adequate.

        See Also:
        ReprojectionOpImage, ReprojectionDescriptor

        Sets a reference to a new reprojected version of the ImageChip base image. For clearing this cache to force it to be recomputed in light of an ImageChip property change, use the method doReprojection().

        NOTE: This method should not need to be called directly. Creating a ReprojectionOpImage via the JAI operation registry with the "cache" flag set will use this method to cache any of its results on the ImageChip. The ReprojectionOpImage should be allowed to manage this caching.

      • doReprojection

        public void doReprojection()
        Clear the reproject image buffer so that the next redraw will force a new reprojection of the chip. This method should be called prior to any changes that will fire a PropertyChangeEvent leading to a repaint being made, such as to a pointing parameter. If it is not called, a new reprojected image will not be created and the property change will have no outwardly visible effect.

        public void setFileName(java.lang.String n,
                        boolean load)
        This method sets the fully qualified file name of the image file from which this chip should retrieve its imagery. If the file is in the VICAR file format, it will also contain additional information used to initialize the ImageChip's PointingModel, CameraModel, and FileModel. If the ImageChip is initialized using a jpl.mipl.mica.util.ImageLoader object, then this image and the various PIG models will be automatically initialized.

        NOTE: The ImageChip must be used in conjunction with a valid image file. It may be initialized without a file, but before it is used, it must have one set through this method.

      • setOperation

        public void setOperation(java.lang.Object[] operation)
        Adds a JAI RenderedOp ParameterBlock to the chip's ImageChain. The ImageChain will handle adding Sources to the ParameterBlock and call the JAI.create() method.

        The operation array block should comprise the following:

        • operation[0]="MyKeyName" - The key name used to retrieve or modify this operation later.
        • operation[1]="JAIOperation" - The string name for the JAI operation to add, delete, or modify.
        • operation[2]=new ParameterBlock() - The new or modified ParameterBlock used to configure the JAI operation.

        This will then cause the chip to fire an "Operation" PropertyChangeEvent with the old and new operation arrays.

      • addPropertyChangeListener

        public void addPropertyChangeListener(java.beans.PropertyChangeListener listener)
        Adds a PropertyChangeListener to be notified when any ImageChip bound properties change. It also adds the listener to the chip's CameraModel, PointingModel, and FileModel. Only one listener may be registered at any one time. Generally, this will be an ImageBag and all tools interested in listening for chip changes should attach a single listener to the ImageBag.
