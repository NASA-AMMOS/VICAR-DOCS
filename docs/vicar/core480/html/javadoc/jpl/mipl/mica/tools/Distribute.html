    public class Distribute
extends javax.swing.JPanel
implements java.beans.PropertyChangeListener
    A Distribute tool is a JavaBean that distributes a pointing parameter change in an anchor ImageChip to other ImageChips. The tool passively listens for PropertyChangeEvents from a PointingModel indicating that a chip has been repointed to correct an error in its parameters. When it receives a change from one of two anchor chips, all chips between those two anchors are moved a percentage of the anchor's correction.

    The set of chips between the anchors is determined by intersecting each chip's border with the tie-line between anchors. If the chip border intersects the line, then it will receive a percentage of the pointing change.

    The amount of pointing change depends on the relative proximity of each chip to the two anchors. The change will be equal to the anchor's motion if it overlaps the near anchor exactly. There will be no change if the distance to the chip from the near anchor is equal to or greater than the far anchor. The change varies linearly with the visual (output pixel) distance between each chip and the near anchor.

    The tool is assisted by a DistributeOverlay object. This subclass of InvisibleOverlay provides the interface with the global output mosaic for determining chip-line intersections and for annotations.

      • propertyChange

        public void propertyChange(java.beans.PropertyChangeEvent evt)
        Handles a change in pointing of an ImageChip, a change in output Projection scale, or a chip being added or removed from the ImageBag. In the latter two cases, the anchors and tie-line graphic are revalidated. In the first case, the change in pointing may be spread to other chips depending on the anchor locations.

        If the chip being repointed (to correct an error in its pointing) is one of the anchors, each chip between this anchor (near) and the other (far) anchor are also moved. This set of chips between the anchors is determined by intersecting each chip's border with the tie-line between anchors. If the chip border intersects the line, then it will receive a percentage of the pointing change.

        The amount of pointing change depends on the relative proximity of each chip to the two anchors. The change will be equal to the anchor's motion if it overlaps the near anchor exactly. There will be no change if the distance to the chip from the near anchor is equal to or greater than the far anchor. The change varies linearly with the visual (output pixel) distance between each chip and the near anchor.
