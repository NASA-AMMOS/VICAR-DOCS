Package jpl.mipl.tlm.jedi Description

JEDI, a moniker given to this application by Allan Runkle, allows for the display of EDRs as they are generated by RealTime. Though intrinsically cool, this application is hopefully useful in that it allows monitoring capabilities of the RealTime output in a Web-based, Java-based, distributed, cross-platform, lightweight, buzz, buzz, buzz-word compliant environment.

A parallel application, called YODA (Your Observable Display Architect, again named by Allan Runkle), allows one to create the Look-and-Feel of the RealTime Display rendered by JEDI. Confused yet?

Originally, JEDI obtained its Look-and-Feel (LnF) from a set of serialized objects created by YODA. These objects, called Personalities, were expected to live in a local directory (default: personalities). Now, you can point JEDI to a URL to obtain these serialized objects.

When this package was first developed, it lived in jpl.mipl.rtdisp. Now, after some use, it is being integrated into the new CORBA RealTime system. Therefore, the package name is now: jpl.mipl.tlm.jedi.
