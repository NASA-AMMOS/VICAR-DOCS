jpl.mipl.mars.viewer.image.config

Class ImageConfiguration

  • java.lang.Object
    • jpl.mipl.mars.viewer.image.config.ImageConfiguration

  • public class ImageConfiguration
extends java.lang.Object
    Purpose: Higher level manager for image types, product contexts, and mission namespaces. Image types are the basic structures binding an image id and name to implementations of the image content, image overlay, and associated properties information. This section is mission independent. Product types are mappings of product ids to their appropriate image type as defined above. A product context is a grouping of these types defined under a particular context id. Mission namespaces bind a mission id to a context id, so that multiple missions can share the same product type conventions. In general, the calling session should maintain an active mission namespace and use this when querying for the image id that is associated with a product id. If the namespace is not used, then a best-effort is made to return an image id only if it is unique amongst all contexts. If not, then an error will be thrown that will include a listing of the potential namespaces, granting the option of user-interaction to select the appropriate namespace from that point.

      • getImageTypeId

        public java.lang.String getImageTypeId(java.lang.String missionNamespace,
                                       java.lang.String productType)
                                throws ImageTypeLookupException
        Requests an image id based on a productType and optional mission namespace. If no namespace is used, then the following logic is used: If there is no productType amongst any context, return null. If there is only one context with that type, use that. If more than one contexts have the type, then the request is considered ambiguous and the associated exception is thrown.
        Parameters:
        missionNamespace - Mission id, optional. Can also be a list string of mission namespace ids
        productType - Product type id
        Returns:
        Image id associated with the parameters.
        Throws:
        ImageTypeLookupException - If no namespace was specified and ambiguous mapping results.
