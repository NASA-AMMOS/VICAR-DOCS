    Plinth - PLaces INTerpolation Helper

    This class interpolates entries between two localized positions. Given an ending site/drive and a view to work in, it looks for the previous entry in that view to use as a starting point (or the starting point may be supplied). It then fetches all the telemetry solutions between those two points, and adjusts them using linear interpolation so that the endpoints match the localized positions, and any error between telemetry and the localized positions is distributed equally along the traverse route.

    The result is not written directly to PLACES but is insted put in an XML file ready for upload. This allows the user to sanity-check the output if desired.

    The output is expressed in the starting Site frame if orbital_frame is -1 (the default), or in the given orbital frame if it's >= 0.

    Author:
    Bob Deen, JPL

      • getPwdFromNetrc

        public static void getPwdFromNetrc(java.lang.StringBuilder passwd,
                                   java.lang.StringBuilder userid,
                                   java.lang.String url_string)
        Attempt to read the .netrc file to get the password token. If it fails for any reason, quietly return null. This is only likely to work on Unix-based machines... on others, the .netrc file will not be found and it will return null. If the userid is null, it'll return the first entry matching the host. Otherwise it'll match the userid. Both the pwd and userid's are StringBuilder objects so we can easily return two strings. They should both be empty when called (or userid set to the username).
