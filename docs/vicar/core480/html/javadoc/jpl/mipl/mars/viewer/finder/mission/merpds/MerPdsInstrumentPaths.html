jpl.mipl.mars.viewer.finder.mission.merpds

Class MerPdsInstrumentPaths

  • java.lang.Object
    • jpl.mipl.mars.viewer.finder.mission.merpds.MerPdsInstrumentPaths

  • public class MerPdsInstrumentPaths
extends java.lang.Object
    Purpose: So...PDS has their own naming convention for the archives...we have Mr Rafael A to thank for making particular decisions. Its messy and should be compartamentalized. However, since we are making remote requests, it would be nice to only request for paths that we care about. Ie if we are looking at Spirit archive, then don't waste time examining the Opportunity paths. 
      File Finder Root Location                           Paths start with ...
  ===============================================     ====================
  http://pdsarchive.nasa.gov/mars/mer/spirit/      ->  'mer2'  
  http://pdsarchive.nasa.gov/mars/mer/opportunity/ ->  'mer1'
  http://pdsarchive.nasa.gov/mars/mer/other/       ->   both
    So, what we do is examine the root location that was passed to the file finder. We assume the last entry in the path is one of {spirit,opportunity}. If it is, then we will create paths only for that mission. If it is NOT however, then the paths returned by an instance of this class will include paths for both rovers. Happy hunting.
