Package jpl.mipl.phx.logging Description

This package contains Log server and related classes for Phoenix but can be used for other projects also.

  • LogServer_IF: Defines the log server interface
  • LogServer : Implementation of LogServer_IF. This server is started by the Phoenix scheduler.
  • PHXLogger : A Logger of type java.util.logging.Logger that the LogServer uses for logging. This class appends log messages to the specified file. The caller of log method on LogServer specifies which logger the message should be logged with. As a matter of fact, the name of the logger is the absolute file name where messages are to be logged.
  • ProxyLogger : A Logger of type java.util.logging.Logger that forwards log messages to the LogServer. Any replacement implementation of the ProxyLogger should make sure that it is Serializable or actually Externalizable. It should also close all the handlers properly.
  • ProxyHandler: A java.util.logging.Handler type the forwards messages to LogServer. ProxyHandler knows which log file the message belongs to and uses this log file name to name the logger. Any replacement implementation of the ProxyHandler should make sure that it is Serializable or actually Externalizable and that it calls LogServer.createLogger and LogServer.removeLogger methods appropriately to allow closing of actual log files. One can append multiple handlers to a single logger which allows, for example, logging same message to two different log files or logging and publishing the same log message, etc.
