Class SirtfDownlinkInfo

  • java.lang.Object
    • jpl.mipl.tlm.sirtf.SirtfDownlinkInfo
  • All Implemented Interfaces:
    java.io.Serializable

    public class SirtfDownlinkInfo
extends java.lang.Object
implements java.io.Serializable
    Collects downlink info that is used to generated the Nominal and Expedited Processing Reports. Some data in this class is written to the DB so that the APEX Status and History displays can be updated. This class is persistent for the purpose of storing the downlink count. The data in an instantiated class is written to an xml file periodically (every 10 DCEs) in the event that the system needs to be restarted or crashes. This class will automatically increment the downlink count when a has completed. A 'get' method is available for this.

      • SirtfDownlinkInfo

        public SirtfDownlinkInfo(boolean state,
                         SirtfEnv env)
        This is the class constructor. It is created when the system is started up initially. All variables will be initialized to their default values, except the downlinkCount. The last stored value (in an xml file) will be read in and incremented. This will be the new downlink count for the next run. This value will incremented when sirtftlmproc determines that the downlink has completed. This bean is restored by reading in the xml file. This is done by the PktStore obj. It will read in the xml file upon initializatioion.
