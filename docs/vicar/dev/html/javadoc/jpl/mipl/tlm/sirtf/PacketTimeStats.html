jpl.mipl.tlm.sirtf

Class PacketTimeStats

  • java.lang.Object
    • jpl.mipl.tlm.sirtf.PacketTimeStats

  • public class PacketTimeStats
extends java.lang.Object
    Accounts for a set of statistics based upon the time (SCLK, SCET, ERT, RCT) of SFDU's processed during a downlink. There are accessor and mutator (getter and setter) methods for every conceivable property, though the mutator's are required by the Java XMLEncode/XMLDecoder mechanism. As such, they should not be used (or be used cautiously) by external objects.

    The available properties are:

    • earliest and latest SCLK (spans exp. and nom. below)
    • earliest and latest SCET (spans exp. and nom. below)
    • earliest and latest ERT (spans exp. and nom. below)
    • earliest and latest RCT (spans exp. and nom. below)
    • earliest and latest expedited SCLK
    • earliest and latest expedited SCET
    • earliest and latest expedited ERT
    • earliest and latest expedited RCT
    • earliest and latest nominal SCLK
    • earliest and latest nominal SCET
    • earliest and latest nominal ERT
    • earliest and latest nominal RCT

    For each of the accessor methods (getters), an aberrant default will be returned if no packets have been seen for that property. These defaults were selected to make Sybase (and probably other databases) happy. Here's the specifics:

    Type default
    earliest {SCET,ERT,ERT} Date (32503680000000)
    3000/01/01-00:00:00.000 UTC
    earliest SCLK 32503680000000
    latest {SCET,ERT,ERT} Date ( 0 )
    1970/01/01-00:00:00.000 UTC
    latest SCLK 0
