    Main program for processing SIRTF telemetry data. This program is currently implemented to process APID's: 13, 15, 16, 20, 21, 23, 30, 31, 40, 41, 51 and 52 (11 and 12 are accepted as well, but should be duplicates of 15 and 16). Other APIDs will be ignored, though their packet information may still be logged if VERBOSE_MODE = ON in the UPF.

    This class has the option of running under the Telemetry Processing System (TPS) or in standalone (without CORBA TPS) mode. Simply supply the -sa switch to obtain standalone functionality.

    Also, for historical (hysterical?) reasons, when running SirtfTlmProc within the TPS, the INSTRUMENT UPF keyword must have a value of CE_PHF. The original SIRTF telemetry processor was written in C++ and only processed the science instrument data. This Java processor was developed to process all other data types and some science data into a different format (ESS) for the field. Over time, it become more easily extensible and overtook the C++ processing responsibilities.
