    SFDU

    public class SirtfSFDU
extends GenericSFDU
    A class that encapsulates a SIRTF SFDU (whether it be a data SFDU, a CDR or an ECDR). At a minimum, a SIRTF SFDU must meet the following conditions:
    • the primary CHDO is of type 2
    • the secondary CHDO is of type 82, and
    • the tertiary CHDO is of type 83.

    However, this class encapsulates three basic SFDU types: data, CDR, or ECDR. For each of these types, the following must also be true for a valid SFDU:
    • for a SIRTF data SFDU
      • the quaternary CHDO does not exist
      • the data CHDO is of type 10
    • for a SIRTF CDR or
      • the quaternary CHDO is of type 27
      • the data CHDO is of type 28
    • for a SIRTF ECDR
      • the quaternary CHDO is of type 32
      • the data CHDO is of type 29

    None of the current SIRTF processing uses CDR's. This class tries to obtain the same information with the getXXX() methods whether or not the source is a data or ECDR SFDU.

      • isSI

        public boolean isSI()
        Flags whether or not this SFDU is for a Science Instrument (in other words, the APID is: 20, 21, 23, 30, 31, 40, 41, 51, or 52). Though not technically science data APIDs, 51 and 52 are from the CE which controls both IRS and MIPS. The data returned in these APID's is formatting similar to those of the actual science instrument APID's. Also, the CTD declares these as being included in the SI APID list.

      • getChannel

        public int getChannel()
        Gets this SFDU's channel (detector) ID. This value is obtained by the following means:
        • for data SFDU: packet header
        • for ECDR:
          • IRAC: channel A-0561
          • IRS: channel S-1109
          • MIPS: inferred

        If the channel cannot be determined, then a default value of 9 will be returned. Note: the reason 9 is used is historical and, even then, is mostly due to the IRS instrument case. The IRS channel is only available in the ancillary packet and/or ECDR of same. The other instruments can have the channel designation determined (directly for IRAC, indirectly for MIPS) within each packet.
