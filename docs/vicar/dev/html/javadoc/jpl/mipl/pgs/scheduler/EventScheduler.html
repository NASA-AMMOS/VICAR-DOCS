          Only one of the following three keywords:
          • "jpl.mipl.pgs.upf.jbpm.processDefinitionFile": A complete path name to a process definition file.
          • "jpl.mipl.pgs.upf.jbpm.processDefinitionName": Name of a process definition as stored in the database. The value is only valid if the keyword is present in the request it's value is "ON".
          • : The ID of a process definition that was stopped or aborted. The value is only valid if the keyword is present in the request it's value is "ON".
          • Following optional values.
          • "jpl.mipl.pgs.upf.perstore": Indicates if the persistent store is in use. If this keyword is missing it is set to OFF.
            • @{value jpl.mipl.pgs.utils.upf.UPF#LOG4J_PROCESS_LOGGER}: An instance of serializable version of Logger object (Use ProxyLogger) that is the logger to be used by the pipeline @{value jpl.mipl.pgs.utils.upf.UPF#LOG4J_PROCESS_LOGGER_FILE_NAME}: The file name (logger name) of the logger to be used by the pipeline.
          • For logger use one of the following:
          • to specify the logger object to be used by request. OR
          • to specify the log file name to be used to create a logger with log server. In this case, the system property must be set to "yes".
